# Document editor

## **Overview**

The Symper App Document Editor is used to create a bundle of documents used for displaying and recording simple and complex data. It is the primary means to construct a form using simple drag-and-drop commands. The document is used by other elements such as cases and tasks to interact with the user. In other words, the Symper document could be used alone to store business information or attached to the Symper business workflow to control data and work systematically.

**The Symper Document Editor interface is roughly divided into four areas:**

* The tool bar on top, below the navigation bar
* The palette on the left side
* The canvas in the middle
* The attribute bar on the right side

### **Controls**

![](<../../.gitbook/assets/image (87).png>)![](<../../.gitbook/assets/image (28) (2).png>)\*\*\*\*

Symper Document provides numerous pre-defined controls which enable you to build rich, beautiful documents. There is a wide range of controls divided into different groups to be used in the configuration of the document.

**Symper control list is divided into 5 groups based on the purpose of each:**

* Display: These controls display content. Images and data tables belong in this group.
* Data input: These controls allow users to enter data directly. A typical example is a text field or a date picker.
* Layout: These controls allow users to better structure the document. Tabs/pages belong in this group.
* Report: These controls allow users to visualize data as a report. Dataflow belongs to this group.
* Action: These controls allow users to take actions to the form.

#### **a. Display**

* Label: Used for displaying only, cannot add input into label control. All types of data could be displayed in label control: text, number, date, decimal,...
* QR code: Used to show QR code for scanning. Control QR is able to convert a charset and display it in the form of QR code. Scan QR code machine then could scan and return the original input charset.
* Image: Used for storage, displaying, and uploading images to the platform.
* Title: Used to display a title of another control in the same document.

#### **b. Data input**

* Text input: To input data and display text data.
* Rich text: To input data and display text data like text input, however this control may store, format and display multiple lines instead of a single line that text input offers. It is intended for longer text. Naturally, this control’s validation is more limited since longer text usually does not follow a specific pattern.

**Tip: For short text, you may consider using the text input control. For numbers, use number control.**

* Number: To capture integer values that are whole numbers, calculate and display numbers. Numbers can be both positive and negative.
* Date: To input data and display date. It offers a convenient way to select a date when submitting documents. (Format for displaying date in Symper platform: dd/mm/yyyy, format in database: yyyy/mm/dd)
* Date time: To input data and display date and time. It offers a convenient way to select a date and optionally a time. (Format for displaying date and time in Symper platform: dd/mm/yyyy hh:mm:ss, format in database: yyyy/mm/dd hh:mm:ss)
* Time: To input data and display time. offers a convenient way to select a time.
* Month: To input data and display month.
* Select: To input data, however, users are allowed to select only 1 value from the list of values.
* Combobox: To input data, users are allowed to search and select multiple values from the list of values.
* Department: To get name of the department from orgchart for displaying.
* Document select:
* Control approval history: used to display the history of task approval, display the specific time to make the approval, the person who did it.
* Email:
* Currency:
* Radio: It provides a choice between two or more options. Single selection like control select but in different forms of display.
* Checkbox: It indicates a choice between the two Boolean values: true and false. If the checkbox is ticked or selected, the stored value is true, otherwise false. If the component is marked as Required, it must be ticked (true) for the form to be valid.
* Control tab and page: used to create a web-like framework, integrating many pages, each page contains many tabs
* Percent: Display percentage data type.
* User: To select users from the platform users list and display users' names. This control will save the ID of users.
* Input filter: Multiple selections from a list, the data outcomes will be saved in form of serial type and separated by “,”.

#### **c. Layout**

* Table: Used to input and display data in the form of table. Control table may consist of all other controls but itself - control table. Only 1 control could be in a cell of a table.

#### **d. Report**

* Report: Used to display reports in the form of table. (Other control/field could be chosen as condition for data query)
* Dataflow: To select dataflow for displaying data in documents.
* Approval history: Display time and users who approve or disapprove the document in case the document is attached in workflow and is approved or disapproved.
* Tracking value: Track change history of any control in the document (except control table, controls in layout group) and display in form of a timeline.

#### **e. Action**

* Submit: A control action to save all data in the document and stop inputting data. Besides, Control submit is used to execute a list of formulas in it.
* Reset: To delete all data that had been inputted earlier (used to resubmit data if users input wrong data)
* Draft: To save the document temporarily in draft status, which the data in that document will not affect the real data. Users may change status from draft to normal status if needed.

In Symper platform, in order to create a form for submitting data, documents abbreviated as doc are needed. The documents may consist of several controls - data input or table - used for submitting data, viewing data, or combining many other demands.
